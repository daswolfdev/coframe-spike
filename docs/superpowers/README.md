# Specs & Plans

The paper trail behind each PR: a design spec (brainstormed, then reviewed
under [SPEC-REVIEW.md](../SPEC-REVIEW.md)) and the implementation plan built
from it. One line per artifact, newest first — the doc-reachable gate makes
unlisted artifacts fail `make check`, so listing here is not optional.

**Specs and plans are point-in-time records.** Once their PR merges they are
history, not living docs — later changes update [design.md](../design.md),
the service READMEs, and the code, never old specs. When a spec and design.md
disagree, design.md wins.
